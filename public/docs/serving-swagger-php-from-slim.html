Serving Swagger-PHP from Slim

For structuring controllers and services, I will be referring to Project structure

Downloading dependencies

  • Install swagger-php via composer to your project composer require zircote/swagger-php

  • Install doctrine annotations composer require doctrine/annotations

  • Download Swagger-UI here

Prepare swagger ui

  • After you have downloaded swagger ui, copy everything from dist to public/swagger/ dir in your project

  • Navigate to public/swagger/swagger-initializer.js and change default url to your desired .json file

Note: For this example, I will call my json doc openapi.json and I will put it in swagger/ dir

window.onload = function() { //<editor-fold desc="Changeable Configuration Block"> // the following lines will be replaced by docker/configurator, when it runs in a docker-container window.ui = SwaggerUIBundle({ url: "./openapi.json", dom_id: '#swagger-ui', deepLinking: true, presets: [ SwaggerUIBundle.presets.apis, SwaggerUIStandalonePreset ], plugins: [ SwaggerUIBundle.plugins.DownloadUrl ], layout: "StandaloneLayout" }); //</editor-fold> };

Organizing code

  • First, we need to organize our code, so we can later know where our annotations and base controllers are

  • Let's create a class that will be responsible for dispatching requests in our app under src/Controllers

<?php declare(strict_types=1); namespace Controllers; class WebAPIController { public function __construct() {} }

  • Now let's register this namespace under psr-4 autoloader in composer.json

{ "autoload": { "psr-4": { "Controllers\\": "./src/Controllers" } } }

Note: refer to SwaggerPHP step by step

  • Now dump autoloader to register new class

composer dump autoloader

OpenAPI Generation

  • We can add the first route to our web api controller that will be responsible for generating open api docs

use OpenApi\Generator as Generator; class WebAPIController { public function __construct() {} public function generateDocs(Request $request, Response $response): Response { $openapi = Generator::scan(['../src'])->toJson(); $file = fopen("../public/swagger/openapi.json", "wa+"); fwrite($file, $openapi); fclose($file); $response->getBody()->write(file_get_contents("../public/swagger/openapi.json")); return $response ->withHeader('Content-type', 'application/json'); } }

  • Now we have to register this route in app/config/routes.php

use Slim\App as Slim; use Controllers\WebAPIController as API; return function (Slim $app) { //... $app->get("/getDoc", [API::class, 'generateDocs']); //... }

Annotations

  • Now, we can set up annotations

/** * @OA\Info( * title="Inventory management system API", * version="1.1.0", * description="Complete API documentation for web based invetory system <br/> Documentation is based on `OAS 3.0` and API is running on `PHP 8.2`", * @OA\Contact( * name="Developer", * email="example@gmail.com" * ), * @OA\License( * name="MIT", * url="https://opensource.org/license/mit" * ), * ) * @OA\Server( * url=http://my-backend.com, * description="Apache" * ) * * @OA\SecurityScheme( * securityScheme="bearerAuth", * type="http", * scheme="bearer", * bearerFormat="JWT", * description="Enter the Bearer Authorization string as follows: `Bearer Generated-JWT-Token`" * ) * */ class WebAPIController { /* my code here */ }

Note: All the annotations will be in WebAPIController along with dispatch methods
Do not separate annotations from their methods

For more reference about setting up swagger with php, read SwaggerPHP step by step For more reference about annotations visit SwaggerPHP annotations

  • That's it, when you visit http://my-backend.com/getDoc you should get openapi.json as a response

  • Now it's up to you how you will handle that with front and show it on swagger-ui
